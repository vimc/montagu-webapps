Further guidance for generating and uploading stochastic runs

We require multiple model runs for each scenario, each of which is based on a random sample from the joint uncertainty distribution (e.g., posterior distribution) of the input parameters. As we want to compare the runs across scenarios to calculate the impact, the same parameter samples should be used across all scenarios, and the runs labelled to ensure we can identify them.

Please supply the parameter values for each sample in the format specified in the parameter template file:

  • One row per parameter sample
  • The first column "run_id" should label the parameter set and the same label used in the burden template files for each scenario
  • The file should contain one further column for each parameter that is varied in the stochastic simulations. Please adapt the column headings with the parameter name.

Both the central and stochastic burden templates are identical for all scenarios. Please generate one copy for each of your scenarios, filling in the required burden outputs. The stochastic burden templates contain a column "run_id" that should indicate which of the parameter sets specified in the parameter file underlies the particular run.

In the test run we require only 20 independent simulations for each scenario.

Montagu does not currently provide facilities for uploading stochastic simulations, therefore we would ask you to upload the stochastic burden files for each scenario as well as the parameter file to a dropbox folder. We will share the link to your modelling group's dropbox folder via email.
